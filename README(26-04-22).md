# VLM-Navigation-Prototype

基于 Habitat-Sim 与 Qwen2-VL 的视觉-语言导航（VLN）原型系统  
**日期**: 2026-04-22  
**作者**: nuaawzh（探测制导与控制技术）

---

## 1. 项目简介

本项目是保研外校导师建议的 VLA/VLM/VLN 方向的入门实践。目标是验证**大视觉语言模型（VLM）在仿真室内环境中的导航决策能力**。

核心闭环：
```
Habitat-Sim 第一视角图像  →  Qwen2-VL-7B 推理  →  导航动作（前/左/右/停）
```

### 关键特性
- 双环境隔离：`habitat`（Python 3.9）负责仿真，`vlm`（Python 3.10）负责推理
- 4-bit 量化 7B 模型，在 RTX 4060 Laptop（8GB）显存上可跑
- 模糊动作解析，容错模型拼写错误（如 `move_foreward`）
- 历史上下文 Prompt + 重复后处理，避免动作死循环

## 5. 快速开始

### Step 1：运行 Habitat 仿真，生成第一视角图像

```bash
conda activate habitat
cd habitat-vlm
python loop-habitat.py
```

输出：
- `frame_000.jpg` ~ `frame_006.jpg`（7 帧连续画面）
- 终端打印每帧的 Agent 位置坐标

cd ..

输出：

- `vlm_actions.txt`：动作序列（`move_forward` / `turn_left` / `turn_right` / `stop`）

## 6. 实验结果与发现

### 6.1 模型对比：2B vs 7B

| 能力 | Qwen2-VL-2B | Qwen2-VL-7B | 结论 |
|------|-------------|-------------|------|
| 场景描述 | 能识别走廊、门、水瓶 | 更精确，幻觉少 | 7B 明显更稳 |
| 物体识别 | 误认"鸡蛋包装盒" | 正确识别"矿泉水瓶" | 7B 细节更好 |
| 动作决策 | 回避回答，输出安全提示 | **直接输出动作** | 7B 可用 |
| 指令遵循 | 经常废话 | 基本遵循 | 7B 更听话 |
| 显存占用 | ~1.5 GB | ~5.5 GB | 8GB 显卡可跑 |

### 6.2 场景对比：城堡 vs 公寓

| 场景 | VLM 表现 | 原因分析 |
|------|---------|---------|
| `skokloster-castle` | 全部输出 `turn_left` | 古建筑纹理复杂，VLM 训练数据以真实室内为主，对仿真城堡泛化差 |
| `apartment_1` | 动作多样，有前进/左转 | 现代公寓与训练数据分布接近，理解更准确 |

**结论**：VLM 对**合成场景的泛化能力**是实际部署的瓶颈，需要结合 Sim2Real 或域适应。

### 6.3 Prompt 工程影响

| Prompt 类型 | 输出特征 |
|-------------|---------|
| 无目标（纯探索） | 偏好原地打转，连续左转 |
| 目标导向（"找到客厅"） | 倾向于 `move_forward`，但缺少障碍物感知 |
| 加历史上下文 | 减少重复，但后处理不能过度干预 |

### 6.4 关键问题发现

1. **单帧无记忆**：VLM 每帧独立推理，不知道之前走过哪里，容易循环
2. **深度缺失**：仅凭 RGB 无法判断前方障碍物距离，导致该停不停
3. **动作拼写不稳定**：7B 会输出 `move_foreward`、`move Forward` 等变体，需模糊匹配
4. **分辨率敏感**：原图 3072×4096 缩到 560 以下会丢失远处细节，但 8GB 显存无法开更高

## 7. 核心代码说明

### 7.1 Habitat 仿真（`habitat-vlm/loop-habitat.py`）

- 加载 `apartment_1.glb` 场景
- 配置 RGB 相机（640×480，高度 1.5m）
- 内置动作：`move_forward`、`turn_left`、`turn_right`
- 注意：`turn_left`/`turn_right` 只改变朝向，不改变位置

### 7.2 VLM 推理（`vlm_nav.py`）

- 4-bit 量化加载 Qwen2-VL-7B
- 图片缩放至 560px 最大边（对齐 28 的倍数）
- Prompt 包含：目标描述 + 历史动作（最近 3 步）
- 模糊匹配解析：支持 `forward`/`foreward`/`forword` 等拼写变体
- 后处理：仅当连续 3 次 `turn_left` 或 `stop` 时才强制切换动作
